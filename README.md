# WordPress Bridge for Home Assistant

Home Assistant custom integration that connects directly to a WordPress site running the companion Home Assistant Bridge plugin.

Home Assistant initiates all network traffic:

- Pushes selected entity states to WordPress.
- Polls WordPress for queued commands.
- Executes allowed commands locally.
- Acknowledges command success or failure back to WordPress.

This avoids exposing Home Assistant publicly and does not require a middle service.

## Install With HACS

1. Open HACS in Home Assistant.
2. Open the three-dot menu.
3. Choose `Custom repositories`.
4. Add this repository URL:

```text
https://github.com/gnoric/homeassistant-wordpress-bridge
```

5. Select category `Integration`.
6. Install `WordPress Bridge`.
7. Restart Home Assistant.
8. Go to `Settings > Devices & services > Add integration`.
9. Search for `WordPress Bridge`.

## Manual Install

Copy this directory into your Home Assistant config directory:

```text
custom_components/wordpress_bridge
```

The final structure should be:

```text
/config/custom_components/wordpress_bridge/
  __init__.py
  api.py
  config_flow.py
  const.py
  manifest.json
  translations/en.json
```

Restart Home Assistant, then add the integration from `Settings > Devices & services`.

## Configuration

The setup form asks for:

- WordPress site URL, for example `https://example.com`
- Bridge API token generated by the WordPress plugin
- Entities to expose
- Command poll interval in seconds
- Whether to push current states on startup

## Companion WordPress Plugin

The WordPress site must expose these REST endpoints:

```text
GET  /wp-json/ha-bridge/v1/status
POST /wp-json/ha-bridge/v1/states
GET  /wp-json/ha-bridge/v1/commands/pending
POST /wp-json/ha-bridge/v1/commands/{id}/ack
```

All Home Assistant requests use:

```text
Authorization: Bearer <bridge-token>
```

## Allowed Commands

Commands are rejected unless the entity is configured in this integration and the service is allowed for its domain:

- `switch`: `turn_on`, `turn_off`, `toggle`
- `light`: `turn_on`, `turn_off`, `toggle`
- `input_boolean`: `turn_on`, `turn_off`, `toggle`
- `fan`: `turn_on`, `turn_off`, `toggle`
- `cover`: `open_cover`, `close_cover`, `stop_cover`
