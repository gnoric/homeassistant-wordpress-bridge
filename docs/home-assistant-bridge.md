# Home Assistant WordPress Bridge

This folder contains the Home Assistant side of the direct WordPress bridge.

## Install

Copy this directory into your Home Assistant config directory:

```text
custom_components/wordpress_bridge
```

Restart Home Assistant, then add the integration from:

```text
Settings > Devices & services > Add integration > WordPress Bridge
```

The setup form asks for:

- WordPress site URL, for example `https://example.com`
- Bridge API token generated by the WordPress plugin
- Entity IDs to expose, comma separated
- Command poll interval in seconds
- Whether to push current states on startup

## Network Model

Home Assistant initiates every network request.

- State updates: Home Assistant posts selected entity states to WordPress.
- Commands: WordPress queues commands, and Home Assistant polls for them.
- Execution: Home Assistant runs the command locally, then acknowledges success or failure to WordPress.

This avoids a public Home Assistant endpoint and avoids a middle service.

## WordPress REST Contract

The companion WordPress plugin should expose these endpoints under:

```text
/wp-json/ha-bridge/v1
```

### GET `/status`

Used during setup and startup.

Response:

```json
{
  "ok": true,
  "name": "Example Site"
}
```

### POST `/states`

Home Assistant pushes one or more states.

Request:

```json
{
  "states": [
    {
      "entity_id": "sensor.living_room_temperature",
      "state": "21.4",
      "attributes": {
        "friendly_name": "Living room temperature",
        "unit_of_measurement": "°C"
      },
      "last_changed": "2026-07-06T12:00:00+00:00",
      "last_updated": "2026-07-06T12:00:00+00:00",
      "context_id": "..."
    }
  ]
}
```

Expected response:

```json
{
  "ok": true,
  "updated": 1
}
```

### GET `/commands/pending?limit=10`

Home Assistant polls this endpoint.

Response:

```json
{
  "commands": [
    {
      "id": 123,
      "entity_id": "switch.garden_lights",
      "command": "turn_on",
      "payload": {},
      "expected_state": "off",
      "expected_context_id": "...",
      "expected_last_updated": "2026-07-06T12:00:00+00:00"
    }
  ]
}
```

The integration currently allows these service commands:

- `switch`: `turn_on`, `turn_off`, `toggle`
- `light`: `turn_on`, `turn_off`, `toggle`
- `input_boolean`: `turn_on`, `turn_off`, `toggle`
- `fan`: `turn_on`, `turn_off`, `toggle`
- `cover`: `open_cover`, `close_cover`, `stop_cover`

Commands for entities not configured in the integration are rejected.

### POST `/commands/{id}/ack`

Home Assistant acknowledges the result.

Success request:

```json
{
  "status": "done",
  "message": "Command executed",
  "state": {
    "entity_id": "switch.garden_lights",
    "state": "on",
    "attributes": {},
    "last_changed": "2026-07-06T12:00:01+00:00",
    "last_updated": "2026-07-06T12:00:01+00:00",
    "context_id": "..."
  }
}
```

Stale command request:

```json
{
  "status": "stale",
  "message": "Stale command skipped: Home Assistant context changed before execution",
  "state": {
    "entity_id": "switch.garden_lights",
    "state": "on",
    "attributes": {},
    "last_changed": "2026-07-06T12:00:01+00:00",
    "last_updated": "2026-07-06T12:00:01+00:00",
    "context_id": "..."
  }
}
```

Failure request:

```json
{
  "status": "failed",
  "message": "Entity 'switch.private_switch' is not exposed by this bridge"
}
```

Expected response:

```json
{
  "ok": true
}
```

## Authentication

All endpoints should require:

```text
Authorization: Bearer <bridge-token>
```

The token should be generated by WordPress and pasted into the Home Assistant integration setup form.
